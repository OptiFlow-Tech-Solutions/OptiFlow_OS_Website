# ACC-001: aria-hidden on Decorative SVGs
- **Status:** Pending | **Priority:** MEDIUM | **Est. Hours:** 1.5
- **Category:** Accessibility | **Phase:** 05
- **Score Impact:** A11y +5

## Problem / Expected / Affected
Over 100 inline SVGs across the site serve a purely decorative purpose (icons, dividers, background flourishes) but lack `aria-hidden="true"`. Screen readers announce each one as an unnamed graphic, creating significant auditory noise for assistive-technology users navigating the site.

Expected: Every decorative SVG carries `aria-hidden="true"`. Meaningful SVGs (logos, charts, diagrams with informational value) retain or receive appropriate accessible labels.

## Acceptance Criteria
- [ ] Zero decorative SVGs without `aria-hidden="true"` across all pages
- [ ] axe-core automated scan reports zero `aria-hidden`–related violations
- [ ] All 14 pages pass accessibility audit for this rule
- [ ] Logos and charts retain accessible labels (no regression)

## Implementation
Bulk add `aria-hidden="true"` to all decorative inline SVGs across all 14 pages, `nav.html`, and `footer.html`. Identify decorative vs. meaningful SVGs by context — icons, dividers, and ornamentation are decorative; the logo and any data charts are meaningful and should keep or receive labels.

## Dependencies / Definition of Done
- **Depends on:** None
- **DoD:** axe-core scan across all 14 pages shows zero serious/critical violations related to SVG accessibility.
