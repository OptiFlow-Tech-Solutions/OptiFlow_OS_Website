# ACC-002: Skip-to-Content Consistency
- **Status:** Pending | **Priority:** MEDIUM | **Est. Hours:** 0.5
- **Category:** Accessibility | **Phase:** 05
- **Score Impact:** A11y +3

## Problem / Expected / Affected
A `.skip-link` element exists on some pages but has not been verified across all 14 pages. Pages without a skip link force keyboard users to tab through the entire navigation on every page load before reaching main content. Inconsistent presence also signals a broken accessibility pattern to audit tools.

Expected: Every page has a working skip link that targets `#main-content`. On first Tab, the skip link becomes visible and focused; activating it moves focus to the main content area.

## Acceptance Criteria
- [ ] Every one of the 14 pages includes a `.skip-link` element targeting `#main-content`
- [ ] Pressing Tab on page load focuses the skip link as the first interactive element
- [ ] Skip link is visually hidden until focused, then appears
- [ ] Activating the skip link correctly moves focus to the main content area

## Implementation
Audit all 14 pages in `src/pages/` for the presence and correctness of the skip link. Add the missing `.skip-link` element to any page that lacks it. Verify keyboard navigation behaviour on each page in a browser.

## Dependencies / Definition of Done
- **Depends on:** None
- **DoD:** All 14 pages pass a manual keyboard-navigation test (Tab → skip link visible → Enter → focus lands on main content).
