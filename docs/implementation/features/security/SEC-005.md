# SEC-005: JWT Rotation Documentation
- **Status:** Pending | **Priority:** MEDIUM | **Est. Hours:** 0.3
- **Category:** Security | **Phase:** 08
- **Score Impact:** Security +2, Docs +3

## Problem
`JWT_SECRET` lives in an environment variable but there is no documented rotation procedure. If the secret is ever leaked, operators have no runbook for remediation.

## Expected
A documented rotation procedure in `.env.example` comments so any operator can rotate the secret without tribal knowledge.

## Affected
- `.env.example`

## Acceptance Criteria
- [ ] Rotation steps documented in `.env.example` comments: generate new secret, update env var, restart Workers, verify JWT validation.
- [ ] Comment includes a command to generate a secure secret (e.g., `openssl rand -hex 64`).
- [ ] Comment notes that all existing tokens become invalid on rotation.
- [ ] Comment warns never to commit the actual secret value.

## Implementation
Add a commented block to `.env.example` explaining the rotation procedure. No code changes required — purely documentation.

## Dependencies
None.

## Definition of Done
- [ ] `.env.example` updated with rotation procedure.
- [ ] Reviewed by at least one team member.
