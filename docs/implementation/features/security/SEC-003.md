# SEC-003: Input Sanitization
- **Status:** Completed | **Priority:** HIGH | **Est. Hours:** —
- **Category:** Security | **Phase:** 03
- **Score Impact:** Security +6

## Problem
User-supplied string fields were stored and emailed without sanitization, creating a stored-XSS vector in admin dashboards and email clients.

## Expected
HTML entity encoding (`&`, `<`, `>`, `"`, `'`) applied to all string fields before storage in KV and before email dispatch.

## Affected
- `form-submit.js` — sanitization before KV write.
- `email.js` — sanitization before email body construction.

## Acceptance Criteria
- [x] All string fields are entity-encoded: `&` → `&amp;`, `<` → `&lt;`, `>` → `&gt;`, `"` → `&quot;`, `'` → `&#39;`.
- [x] Sanitization runs before KV `put()`.
- [x] Sanitization runs before Resend email dispatch.
- [x] Valid UTF-8 characters pass through unmodified.

## Implementation
A shared `sanitize(str)` utility function walks the five target characters and replaces them with their HTML entity equivalents. Called in both `form-submit.js` (KV path) and `email.js` (email path).

## Dependencies
None.

## Definition of Done
- [x] Deployed to production.
- [x] Unit test confirms all five entities are encoded.
- [x] Fuzz test with random strings produces no raw HTML tags in output.
