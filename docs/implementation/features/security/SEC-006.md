# SEC-006: CSP Meta Tag in HTML Head
- **Status:** Pending | **Priority:** MEDIUM | **Est. Hours:** 0.3
- **Category:** Security | **Phase:** 10
- **Score Impact:** Security +4

## Problem
Content-Security-Policy is set only at the nginx/netlify HTTP header level. Adding an HTML `<meta>` tag provides defense-in-depth and works even when headers are stripped or misconfigured.

## Expected
Every page includes a `<meta http-equiv="Content-Security-Policy">` tag that mirrors the HTTP header policy, allowing self-origin scripts/styles plus recognized third-party domains.

## Affected
- `assemble.mjs` (head injection for all pages)

## Acceptance Criteria
- [ ] CSP meta tag present on all generated pages.
- [ ] Allows: `'self'`, `fonts.googleapis.com`, `fonts.gstatic.com`, `plausible.io`, `calendly` (via `BE-001`).
- [ ] Blocks inline scripts except those with a nonce or hash.
- [ ] Does not break existing fonts, analytics, or form submission.

## Implementation
Add the CSP `<meta>` tag to the shared `<head>` injection logic in `assemble.mjs`. The value should be a semicolon-delimited policy string covering `default-src`, `script-src`, `style-src`, `font-src`, and `connect-src`.

## Dependencies
- `BE-001` — must allow Calendly domain after Calendly integration is complete.

## Definition of Done
- [ ] CSP meta tag appears on all 14 pages.
- [ ] `npm run validate` passes.
- [ ] No console CSP violation errors on production.
