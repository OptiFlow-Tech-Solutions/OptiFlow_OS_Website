# SEC-002: CSRF Protection
- **Status:** Completed | **Priority:** HIGH | **Est. Hours:** —
- **Category:** Security | **Phase:** 03
- **Score Impact:** Security +5

## Problem
Form submission endpoints accepted cross-origin requests from any domain, making them vulnerable to CSRF attacks.

## Expected
Origin/referer header validation against an allowlist of permitted domains. Requests from disallowed origins receive HTTP 403.

## Affected
- `form-submit.js`

## Acceptance Criteria
- [x] Allowed origins: `optiflow.in`, `os.optiflow.co.in`, `www.optiflow.in`.
- [x] Requests with disallowed `Origin` or `Referer` headers return 403.
- [x] Missing origin header is handled permissive (not blocked) for backwards compatibility.

## Implementation
A middleware check in the Worker inspects the `Origin` and `Referer` headers against the allowlist before processing form payloads. Returns `403 Forbidden` with a descriptive JSON error body for mismatches.

## Dependencies
None.

## Definition of Done
- [x] Deployed to production.
- [x] E2E form-submission tests pass from allowed origins.
- [x] Cross-origin requests from unauthorized domains are rejected.
