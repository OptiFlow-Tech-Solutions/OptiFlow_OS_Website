# SEO-001: Product + SoftwareApplication Structured Data
- **Status:** Pending | **Priority:** MEDIUM | **Est. Hours:** 0.5
- **Category:** SEO | **Phase:** 09
- **Score Impact:** SEO +3

## Problem / Expected / Affected
Missing Product and SoftwareApplication JSON-LD on the homepage and product pages means search engines lack structured understanding of what OptiFlow OS is and offers. This directly impacts rich-result eligibility in SERPs.

Expected: Product schema on the homepage with name, description, and pricing; SoftwareApplication schema on the features/product page with feature list and OS category classification.

## Acceptance Criteria
- [ ] Homepage includes `Product` JSON-LD with `name`, `description`, and `offers` (pricing)
- [ ] Product page includes `SoftwareApplication` JSON-LD with `applicationCategory`, `operatingSystem`, and `featureList`
- [ ] All schema validates against schema.org via the Rich Results Test

## Implementation
Extend `injectJSONLD()` in `assemble.mjs` to emit both `Product` and `SoftwareApplication` schemas using data already present in `site.json`. No new dependencies required — all values are static.

## Dependencies / Definition of Done
- **Depends on:** None
- **DoD:** Both schema types present and validating on target pages; no hardcoded company info (use site.json).
