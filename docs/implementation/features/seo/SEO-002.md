# SEO-002: Dynamic FAQPage Structured Data
- **Status:** Pending | **Priority:** MEDIUM | **Est. Hours:** 1.0
- **Category:** SEO | **Phase:** 09
- **Score Impact:** SEO +3

## Problem / Expected / Affected
The `FAQPage` JSON-LD is hardcoded with only 5 Q&A pairs in `assemble.mjs`, while the `faq.html` source contains many more actual questions. Search engines see an incomplete FAQ schema that doesn't reflect the page content, which can hurt rich-snippet eligibility and cause content/schema mismatch penalties.

Expected: At build time, parse `faq.html` to extract every `.faq-question` and `.faq-answer` text node, then generate a complete `FAQPage` JSON-LD block reflecting all questions on the page.

## Acceptance Criteria
- [ ] All `.faq-question` and `.faq-answer` text extracted from `faq.html` at build time
- [ ] `FAQPage` JSON-LD includes every extracted Q&A pair
- [ ] Schema validates via Rich Results Test
- [ ] No hardcoded Q&A data remains in `assemble.mjs`

## Implementation
In `assemble.mjs`, read `faq.html` source, parse the DOM (using the existing DOM parser used elsewhere in the build), query all `.faq-question` / `.faq-answer` pairs, and emit them into the `FAQPage` JSON-LD schema dynamically.

## Dependencies / Definition of Done
- **Depends on:** None
- **DoD:** FAQ page schema matches actual page content exactly; build-time extraction, not manual sync.
