# API-001: API Versioning Prefix
- **Status:** Pending | **Priority:** MEDIUM | **Est. Hours:** 1.0
- **Category:** API | **Phase:** 08
- **Score Impact:** API +5

## Problem
API endpoints are served directly at `/api/form-submit` with no version prefix. Any breaking change to the contract would silently break existing clients.

## Expected
All API routes live under `/api/v1/`. Old routes either redirect (301) or return 410 Gone so clients can detect the break and upgrade.

## Affected
- `form-submit.js` (Worker route handling)
- `core.js` (client-side `fetch` URLs)

## Acceptance Criteria
- [ ] `POST /api/v1/form-submit` accepts submissions.
- [ ] `POST /api/form-submit` returns 301 redirect to `/api/v1/form-submit`.
- [ ] `core.js` fetch calls updated to use `/api/v1/*` URLs.
- [ ] E2E form-submission tests pass against new endpoints.
- [ ] Backwards-compatible redirect preserves POST body.

## Implementation
Update the Cloudflare Worker route matcher to handle both `/api/v1/*` and legacy `/api/*` paths. Inject a redirect handler that issues 301 with the new path. Update the `core.js` endpoint constant to point at `/api/v1/form-submit`.

## Dependencies
None.

## Definition of Done
- [ ] New routes deployed and functional.
- [ ] Redirects return correct status codes.
- [ ] All existing tests pass.
- [ ] `npm run validate` passes.
