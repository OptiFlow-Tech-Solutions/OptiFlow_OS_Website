# BE-001: Calendar Integration (Calendly)
- **Status:** Pending | **Priority:** HIGH | **Est. Hours:** 2.0
- **Category:** Backend | **Phase:** 08
- **Score Impact:** Product +10

## Problem
The demo booking calendar on `demo-booking.html` is fake — pure HTML/CSS with no backend. Every booking silently fails, creating a broken user experience and leaking leads.

## Expected
A real Calendly (or Cal.com) inline widget embedded on the demo booking page, with prefill data from the form fields and UTM tracking.

## Affected
- `demo-booking.html`

## Acceptance Criteria
- [ ] Calendly inline widget loads and renders in the calendar section.
- [ ] User can select a date/time slot and complete a booking.
- [ ] Confirmation message displayed after successful booking.
- [ ] Form fields (name, email, phone, company) pre-populate the Calendly invitee fields.
- [ ] UTM parameters passed through to Calendly for attribution.
- [ ] Fake CSS calendar placeholder removed or hidden.

## Implementation
Embed the Calendly inline widget script with a configured event type URL. Use `data-utm-*` attributes and the Calendly prefill API to pass form field values. Remove the decorative CSS-only calendar element.

## Dependencies
None.

## Definition of Done
- [ ] Widget loads on demo-booking page.
- [ ] End-to-end booking flow verified (select slot → fill details → confirm → receive email).
- [ ] Fake calendar UI removed.
- [ ] `npm run validate` passes.
