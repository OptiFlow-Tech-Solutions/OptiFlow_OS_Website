# FE-003: Cookie Consent Banner (GDPR/DPDP Act)
- **Status:** Pending | **Priority:** HIGH | **Est. Hours:** 1.5
- **Category:** Compliance | **Phase:** 10
- **Score Impact:** Product +5, Security +3

## Problem
No cookie consent despite Plausible analytics. Indian DPDP Act 2023 and GDPR require opt-in consent before tracking.

## Expected
A cookie consent banner that gates analytics loading behind explicit user consent, stored in localStorage.

## Affected
- **Pages:** All 16
- **Files:** src/partials/cookie-banner.html (new), core.css, core.js

## Acceptance Criteria
- [ ] Banner appears on first visit for all pages
- [ ] Accept button enables Plausible analytics
- [ ] Decline button blocks analytics
- [ ] Preference persisted in localStorage
- [ ] Privacy policy linked in banner text

## Implementation
Create a new partial at src/partials/cookie-banner.html with a fixed-bottom banner styled via core.css. Add JS consent logic to core.js that checks localStorage for existing preference, shows banner if absent, and conditionally loads Plausible script only when accepted. Inject the partial into all pages via assemble.mjs.

## Dependencies
UX-004 (style consistency in core.css)

- [ ] Code implemented
- [ ] `npm run validate` passes
- [ ] `npm test` passes
