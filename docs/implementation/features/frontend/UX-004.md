# UX-004: Extract Page Styles to core.css
- **Status:** Pending | **Priority:** MEDIUM | **Est. Hours:** 4.0
- **Category:** Design System | **Phase:** 06
- **Score Impact:** Frontend +5

## Problem
All 14 pages contain 200-600 lines of inline <style> blocks, violating the AGENTS.md convention that component styles belong in core.css.

## Expected
80%+ of inline styles extracted to core.css with BEM naming; zero duplicate CSS across pages.

## Affected
- **Pages:** All 14 pages
- **Files:** core.css, all 14 page files

## Acceptance Criteria
- [ ] 80%+ of inline styles extracted from pages into core.css
- [ ] No duplicate CSS patterns between any two pages
- [ ] All pages render identically after migration
- [ ] `npm run validate` passes without errors

## Implementation
Audit all 14 pages to identify shared patterns (hero, cards, grids, sections, CTAs). Extract each pattern into core.css using BEM naming conventions. Replace inline <style> blocks with the new core.css class references. Verify each page individually by comparing before/after screenshots or rendered output.

## Dependencies
None, but blocks UX-005 and FE-003

## Definition of Done
- [ ] Code implemented
- [ ] `npm run validate` passes
- [ ] `npm test` passes
