# UX-001: Unify Form Component CSS
- **Status:** Pending | **Priority:** MEDIUM | **Est. Hours:** 1.5
- **Category:** Design System | **Phase:** 06
- **Score Impact:** Frontend +3, UI/UX +3

## Problem
Two different form CSS patterns exist: .form-input on one page vs .form-group input on another.

## Expected
A single .form-* BEM naming convention in core.css used identically by all form pages.

## Affected
- **Pages:** contact, demo-booking
- **Files:** core.css, contact.html, demo-booking.html

## Acceptance Criteria
- [ ] Single .form-* BEM naming defined in core.css
- [ ] Both contact and demo-booking pages use identical form class names
- [ ] Dark mode renders form elements consistently
- [ ] No inline form styles remain on either page

## Implementation
Audit the two form implementations to identify the best pattern. Extract common form styles (label, input, textarea, select, submit) into core.css under a single BEM namespace. Update both contact.html and demo-booking.html to use the unified classes. Verify visual parity in light and dark mode.

## Dependencies
None

- [ ] Code implemented
- [ ] `npm run validate` passes
- [ ] `npm test` passes
