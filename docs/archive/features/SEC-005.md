# SEC-005: JWT Secret Rotation Documentation
- **Feature ID:** SEC-005
- **Category:** Security
- **Parent Module:** Security
- **Priority:** MEDIUM
- **Phase:** 08
- **Status:** Pending
- **Est. Hours:** 0.3

## Business Goal
Provide a documented, repeatable process for rotating JWT signing secrets to reduce the impact window of a potential secret leak and align with security best practices.

## Technical Goal
Add rotation instructions and comments to `.env.example` and create a lightweight security runbook documenting the JWT secret rotation procedure.

## Problem Statement
The JWT signing secret is stored in an environment variable with no documented rotation procedure. In the event of a suspected leak, there is no clear process for operators to rotate the secret, which could prolong exposure.

## Current Behavior
JWT secret is read from an environment variable. There are no comments or documentation explaining how or when to rotate it. `.env.example` contains no guidance.

## Expected Behavior
`.env.example` includes explanatory comments about the JWT secret: what it is used for, minimum length/complexity requirements, and a link to the rotation runbook. A `docs/security/jwt-rotation.md` runbook documents the step-by-step rotation process including generating a new secret, updating the environment variable, restarting services, and handling in-flight tokens.

## Implementation Notes

Create `docs/security/jwt-rotation.md` with a concise runbook: generate via `openssl rand -hex 64`, update the env var, restart the application, and note that existing tokens remain valid until expiry (or document forced invalidation if needed).
