# UX-003: Dark Mode Logo in Navigation
- **Feature ID:** UX-003 | **Category:** Frontend | **Priority:** LOW | **Phase:** 06 | **Status:** Pending | **Est. Hours:** 0.2

## Business Goal
Ensure brand visibility is maintained when users browse in dark mode, preventing the logo from blending into a dark background.

## Technical Goal
Add a CSS filter or `<picture>` element fallback so the navigation logo is visible on both light and dark backgrounds.

## Problem Statement
The navigation bar logo has no dark-mode adaptation. Only the footer logo includes `filter: brightness(10)`. When a user's OS or browser is in dark mode, the nav logo (typically a dark-colored SVG/PNG) becomes invisible or poorly visible against dark backgrounds.

## Current Behavior
Navigation logo renders with no color-scheme-aware adjustments. Footer logo has `filter: brightness(10)` applied.

## Expected Behavior
Navigation logo inverts or switches to a light variant when the page is viewed in dark mode, matching the footer behavior.

## Affected Pages / Components / Files
- `assets/css/core.css` — add dark-mode logo filter rule
- `src/includes/nav.html` — optionally add `<picture>` with light/dark sources

## Dependencies
None

## Acceptance Criteria
- [ ] Navigation logo is clearly visible in both light and dark color schemes
- [ ] Approach is consistent with the existing footer logo treatment
- [ ] `npm run build && npm run validate` passes

## Definition of Done
- [ ] Code implemented
- [ ] Tests passing
- [ ] Validation passing

## Implementation Notes
Simplest approach: add `@media (prefers-color-scheme: dark) { .nav-logo { filter: brightness(10); } }` to `core.css`. If a distinct white variant asset exists, use `<picture>` with `prefers-color-scheme` media queries for a crisper result.

## Rollback Notes
Remove the added CSS rule or `<picture>` element.
