# CNT-004: Favicon Variants
- **Feature ID:** CNT-004 | **Category:** Frontend | **Priority:** LOW | **Phase:** 07 | **Status:** Pending | **Est. Hours:** 0.3

## Business Goal
Ensure proper favicon display across all browsers, devices, and contexts (bookmarks, home screens, search results), presenting a polished brand touchpoint.

## Technical Goal
Generate and serve `.ico` and SVG favicon variants alongside the existing PNG favicon, and update the build pipeline to include them in `<head>`.

## Problem Statement
Only a PNG favicon is currently referenced. Some browsers and platforms (older IE, Safari pinned tabs, Android home screen) expect `.ico` or SVG formats. The absence of these variants can result in missing or broken favicon displays in certain contexts.

## Current Behavior
A single `<link rel="icon" type="image/png">` tag references the PNG favicon. No `.ico` or SVG variant is generated or linked.

## Expected Behavior
The `<head>` includes `rel="icon"` links for `.ico`, `.png`, and SVG favicon formats, all sourced from the same base asset. The build pipeline auto-generates missing variants.

## Affected Pages / Components / Files
- `scripts/assemble.mjs` — add favicon generation step, update `<head>` injection

## Dependencies
None

## Acceptance Criteria
- [ ] `.ico` favicon is generated and linked with `rel="icon"`
- [ ] SVG favicon is generated and linked with `rel="icon" type="image/svg+xml"`
- [ ] The existing PNG favicon reference is retained
- [ ] All favicon variants are served from the build output directory
- [ ] `npm run build && npm run validate` passes

## Definition of Done
- [ ] Code implemented
- [ ] Tests passing
- [ ] Validation passing

## Implementation Notes
Use a Node.js image library (e.g., `sharp` or `to-ico`) in `assemble.mjs` to convert the source PNG into `.ico` and SVG formats. Add the generated files to the `dist/` output. Update the `<head>` template to include all three `<link>` tags. If an image processing library is not already a dependency, evaluate whether a zero-dependency approach (e.g., a hand-crafted SVG wrapper referencing the PNG as a data URI) is sufficient.

## Rollback Notes
Remove the favicon generation step and the additional `<link>` tags, leaving only the original PNG reference.
