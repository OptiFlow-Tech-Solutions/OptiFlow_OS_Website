# SEO-002: Dynamic FAQPage Structured Data
- **Feature ID:** SEO-002 | **Category:** SEO | **Priority:** MEDIUM | **Phase:** 09 | **Status:** Pending | **Est. Hours:** 1

## Business Goal
Maximize FAQ rich-result coverage in search results by exposing all Q&A content as structured data, improving the likelihood of appearing in "People also ask" and FAQ carousels.

## Technical Goal
Replace the hardcoded 5-question FAQ JSON-LD with a build-time parser that extracts all question/answer pairs from the FAQ page HTML.

## Problem Statement
The FAQ page contains significantly more than 5 question-and-answer pairs, but only the first 5 are manually defined in the `injectJSONLD` function. The remaining Q&A content is invisible to search engines for structured-data purposes, capping the site's FAQ rich-result potential.

## Current Behavior
`assemble.mjs` injects a static `FAQPage` JSON-LD block with exactly 5 hardcoded `Question`/`Answer` pairs.

## Expected Behavior
During the build process, the FAQ page HTML is parsed to extract all `Question`/`Answer` pairs (identified by consistent markup patterns such as `<dt>`/`<dd>`, `<details>`/`<summary>`, or heading + paragraph combinations), and the full set is emitted as `FAQPage` JSON-LD.

## Affected Pages / Components / Files
- `scripts/assemble.mjs` — add HTML parsing logic to `injectJSONLD`
- `src/pages/faq.html` — ensure consistent Q&A markup for reliable parsing

## Dependencies
None

## Acceptance Criteria
- [ ] FAQ JSON-LD includes every Q&A pair present on the FAQ page, not a fixed subset
- [ ] Parsing works regardless of the number of questions added or removed
- [ ] Generated JSON-LD passes Google Rich Results Test validation
- [ ] No Q&A pairs are duplicated or missing
- [ ] `npm run build && npm run validate` passes

## Definition of Done
- [ ] Code implemented
- [ ] Tests passing
- [ ] Validation passing

## Implementation Notes
Add a `parseFAQContent(html)` function to `assemble.mjs` that uses a DOM parser (e.g., `jsdom` if already a dependency, or regex if the markup is predictable enough). Match on a consistent pattern — the FAQ page should use either `<details><summary>` elements or a structured `<dl>` list for reliable extraction. If the FAQ page markup is inconsistent, standardize it first as part of this feature.

## Rollback Notes
Restore the static 5-question hardcoded FAQPage JSON-LD and remove the parsing function.
