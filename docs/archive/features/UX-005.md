# UX-005: Breadcrumb Navigation
- **Feature ID:** UX-005 | **Category:** Frontend | **Priority:** LOW | **Phase:** 06 | **Status:** Pending | **Est. Hours:** 1.5

## Business Goal
Improve user orientation and SEO by providing clear breadcrumb trails on interior pages, reducing bounce rates and helping search engines understand site structure.

## Technical Goal
Implement breadcrumb navigation using the existing CSS placeholder classes in `core.css`, and auto-generate breadcrumb markup during the build process.

## Problem Statement
`core.css` already contains CSS classes for breadcrumbs (`.breadcrumb`, `.breadcrumb-item`, `.breadcrumb-separator`) but no page includes the corresponding HTML markup. Breadcrumb navigation was designed but never implemented.

## Current Behavior
No breadcrumb navigation appears on any page. The CSS rules exist but are unused.

## Expected Behavior
Each page (except the homepage) displays a breadcrumb trail between the navigation bar and the hero section, derived from the page's position in the site structure defined in `site.json`.

## Affected Pages / Components / Files
- `assets/css/core.css` — review/maybe adjust breadcrumb CSS
- `src/includes/nav.html` — add breadcrumb placeholder or include
- `scripts/assemble.mjs` — auto-generate breadcrumb markup per page

## Dependencies
None

## Acceptance Criteria
- [ ] Breadcrumb trails appear on all interior pages (not homepage)
- [ ] Breadcrumbs are auto-generated from `site.json` page hierarchy, not manually maintained
- [ ] Breadcrumbs use semantic `<nav aria-label="Breadcrumb">` markup with structured data
- [ ] Mobile viewports render breadcrumbs without overflow
- [ ] `npm run build && npm run validate` passes

## Definition of Done
- [ ] Code implemented
- [ ] Tests passing
- [ ] Validation passing

## Implementation Notes
Add a `generateBreadcrumb(pageId)` function to `assemble.mjs` that walks `site.json` pages array to find the parent chain. Inject the resulting `<nav>` into each page during assembly. Use `<!-- INCLUDE: breadcrumb -->` placeholder in page templates. The homepage should omit the breadcrumb entirely.

## Rollback Notes
Remove the breadcrumb include from page templates and the generation function from `assemble.mjs`.
