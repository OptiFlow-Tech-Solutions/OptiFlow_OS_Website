# UX-002: Align Container Width to 1320px
- **Feature ID:** UX-002 | **Category:** Frontend | **Priority:** MEDIUM | **Phase:** 06 | **Status:** Pending | **Est. Hours:** 0.2

## Business Goal
Ensure the website layout matches the approved design specification, avoiding an unintentionally narrow presentation that reduces content density and perceived value.

## Technical Goal
Update the `--container` CSS custom property from `1200px` to `1320px` to match the value specified in `DESIGN.md`.

## Problem Statement
`core.css` defines `--container: 1200px` but `DESIGN.md` specifies a `1320px` max-width for the site container. This 120px discrepancy means all pages render narrower than designed, reducing usable horizontal space and deviating from the approved design.

## Current Behavior
All `.container` elements are constrained to a `1200px` maximum width.

## Expected Behavior
All `.container` elements are constrained to a `1320px` maximum width, matching `DESIGN.md`.

## Affected Pages / Components / Files
- `assets/css/core.css` — update `--container` value

## Dependencies
None

## Acceptance Criteria
- [ ] `--container` CSS variable is set to `1320px`
- [ ] All pages render at the wider container width with no layout breakage
- [ ] `npm run build && npm run validate` passes

## Definition of Done
- [ ] Code implemented
- [ ] Tests passing
- [ ] Validation passing

## Implementation Notes
Single-line change in `core.css`: `--container: 1320px;`. Perform a visual sweep of all 14 pages to confirm no layout regressions (e.g., grid gutters, hero alignment) from the extra 120px.

## Rollback Notes
Revert the value back to `1200px`.
