# SEC-006: CSP Meta Tag in HTML Head
- **Feature ID:** SEC-006
- **Category:** Security
- **Parent Module:** Security
- **Priority:** MEDIUM
- **Phase:** 10
- **Status:** Pending
- **Est. Hours:** 0.3

## Business Goal
Provide a defense-in-depth Content Security Policy layer via HTML meta tag to complement server-level CSP headers, reducing the risk of XSS and data injection attacks.

## Technical Goal
Inject a CSP `<meta>` tag into the `<head>` of all pages via the build script (`assemble.mjs`) to provide an additional layer of CSP enforcement at the document level.

## Problem Statement
Content Security Policy is currently enforced only at the nginx/Netlify server level via HTTP headers. If the server-level CSP is misconfigured or bypassed, there is no document-level fallback, creating a single point of failure for CSP protection.

## Current Behavior
CSP is delivered solely through HTTP response headers configured at the reverse proxy or hosting platform level. No `<meta http-equiv="Content-Security-Policy">` tag exists in the HTML.

## Expected Behavior
Every page includes a `<meta http-equiv="Content-Security-Policy">` tag in the `<head>`, injected during the build process. The meta tag CSP mirrors the server-level policy and serves as a defense-in-depth fallback.

## Affected Pages
- All 14 site pages (index, about, services, contact, careers, blog, pricing, faq, terms, privacy, portfolio, testimonials, 404, sitemap)

## Affected Components
- Build pipeline (`assemble.mjs`)
- CSP meta tag injection logic

## Affected Files
- `scripts/assemble.mjs`

## Dependencies
- SEC-004 (existing CSP header configuration at server level)

## Acceptance Criteria
- [ ] All 14 pages contain a CSP `<meta>` tag in `<head>` after build
- [ ] Meta tag CSP directives mirror the server-level CSP policy
- [ ] CSP meta tag is injected via `assemble.mjs` build script, not manually added to each page
- [ ] Build produces no errors or warnings related to the injection

## Definition of Done
- [ ] Code implemented
- [ ] Tests passing

## Implementation Notes
Add a CSP injection step in `scripts/assemble.mjs` that inserts a `<meta http-equiv="Content-Security-Policy" content="...">` tag into the `<head>` of each assembled page. The CSP directive string should be configurable or read from a shared constant so it stays in sync with the server-level CSP. Inject after `<meta charset>` and before `<title>`.

## Rollback Notes
Remove the CSP meta tag injection step from `assemble.mjs` and rebuild.
