# UX-001: Unify Form Component CSS
- **Feature ID:** UX-001 | **Category:** Design System | **Priority:** MEDIUM | **Phase:** 06 | **Status:** Pending | **Est. Hours:** 1.5

## Business Goal
Provide a consistent form experience across all pages, reducing visual inconsistency that erodes user trust during conversion-critical interactions (contact, demo booking).

## Technical Goal
Eliminate the dual form CSS implementation pattern where `.form-input` classes and `.form-group input` descendant selectors coexist, consolidating into a single shared style in `core.css`.

## Problem Statement
`contact.html` styles its form inputs using standalone `.form-input` classes while `demo-booking.html` targets inputs via `.form-group input` descendant selectors. Both produce visually similar results but differ in specificity, making it impossible to change form appearance in one place. Any design change to forms requires editing two unrelated rule sets.

## Current Behavior
- `contact.html` uses `<input class="form-input">` with rules scoped to `.form-input`
- `demo-booking.html` uses `<div class="form-group"><input></div>` with rules scoped to `.form-group input`
- Styles are embedded in page-level `<style>` blocks

## Expected Behavior
A single set of form component classes defined in `core.css` drives the appearance of all form inputs, textareas, selects, labels, and validation states. Both pages reference the same classes and render identically.

## Affected Pages / Components / Files
- `assets/css/core.css` — add unified form component styles
- `src/pages/contact.html` — replace inline form styles with core.css classes
- `src/pages/demo-booking.html` — replace inline form styles with core.css classes

## Dependencies
None

## Acceptance Criteria
- [ ] All form inputs, textareas, selects, labels, and buttons share a single set of CSS classes defined in `core.css`
- [ ] `contact.html` and `demo-booking.html` produce visually identical forms after the change
- [ ] No form-related styles remain in page-level `<style>` blocks
- [ ] `npm run build && npm run validate` passes

## Definition of Done
- [ ] Code implemented
- [ ] Tests passing
- [ ] Validation passing

## Implementation Notes
Start by extracting the union of all form-related CSS rules from both pages, deduplicating, and placing them in `core.css` under a `/* Forms */` section. Then replace page-level rules with references to the shared classes. Keep existing class names where possible to minimize markup changes.

## Rollback Notes
Revert the `core.css` addition and restore the page-level `<style>` blocks in both HTML files.
