# UX-004: Move Page Styles to core.css
- **Feature ID:** UX-004 | **Category:** Design System | **Priority:** MEDIUM | **Phase:** 06 | **Status:** Pending | **Est. Hours:** 4

## Business Goal
Reduce maintenance burden and enforce design consistency by centralizing all component styles, making future design changes predictable and low-risk.

## Technical Goal
Extract shared component styles from page-level `<style>` blocks into `core.css`, per the AGENTS.md directive: "Never put component styles in page files — add them to `core.css`."

## Problem Statement
All 14 pages contain 200–600 lines of inline `<style>` blocks for components such as hero sections, cards, feature grids, pricing tables, and contact sections. Many of these styles are duplicated across pages. Any global design change requires editing multiple pages, increasing risk of drift.

## Current Behavior
Each page carries its own `<style>` block with component-specific styles. `core.css` holds only foundational styles (variables, reset, typography, layout).

## Expected Behavior
Common component patterns are defined once in `core.css` and referenced by all pages. Only page-specific, truly unique styles remain in page-level `<style>` blocks.

## Affected Pages / Components / Files
- `assets/css/core.css` — recipient of extracted component styles
- All 14 pages in `src/pages/` — source of styles to extract

## Dependencies
None

## Acceptance Criteria
- [ ] Common component styles (hero, card, grid, CTA, section padding, etc.) live in `core.css`
- [ ] Each page-level `<style>` block is reduced to page-unique rules only
- [ ] No visual regressions across any of the 14 pages
- [ ] `npm run build && npm run validate` passes

## Definition of Done
- [ ] Code implemented
- [ ] Tests passing
- [ ] Validation passing

## Implementation Notes
Phase this incrementally: start with the most duplicated patterns (hero, card, CTA button, section spacing). After each extraction, rebuild and visually check the affected pages. Target pages with the most duplication first (index, about, services). Use CSS variables already defined in `core.css` for all values.

## Rollback Notes
Restore extracted styles to their original page-level `<style>` blocks and remove the additions from `core.css`.
