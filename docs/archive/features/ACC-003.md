# ACC-003: ARIA Labels on Interactive Components
- **Feature ID:** ACC-003
- **Category:** Accessibility
- **Parent Module:** Accessibility
- **Phase:** 05
- **Status:** Pending
- **Est. Hours:** 2

## Business Goal
Ensure all interactive elements are properly labeled so assistive technology users can understand and operate every control on the site.

## Technical Goal
Add `role` and `aria-label` attributes to interactive elements (buttons, links, form controls, navigation items) that currently lack them across all 14 pages, nav, and footer.

## Problem Statement
Some interactive elements such as icon-only buttons, toggle controls, and navigation items lack proper ARIA roles and labels. Screen readers announce these as generic elements or fail to describe their purpose, making the site difficult to use.

## Current Behavior
Interactive elements without ARIA labels are announced by screen readers as "button" or "link" without describing their function (e.g., a hamburger menu button announced only as "button").

## Expected Behavior
Every interactive element has an appropriate `role` (where the implicit role is insufficient) and `aria-label` (where visible text is absent). Icon-only buttons and links have descriptive `aria-label` values. Navigation landmarks use `role="navigation"`.

## Affected Pages
- All 14 site pages (index, about, services, contact, careers, blog, pricing, faq, terms, privacy, portfolio, testimonials, 404, sitemap)
- `src/partials/nav.html`
- `src/partials/footer.html`

## Affected Components
- Mobile hamburger menu toggle
- Icon-only buttons (search, cart, social)
- Dropdown toggles
- Carousel/slider controls
- Form submit buttons without visible text
- Navigation landmarks
- Footer link groups

## Affected Files
- `src/pages/*.html` (all 14 pages)
- `src/partials/nav.html`
- `src/partials/footer.html`
- `assets/css/core.css`
- `assets/js/core.js`

## Dependencies
- none

## Acceptance Criteria
- [ ] All icon-only buttons have descriptive `aria-label` attributes
- [ ] Navigation elements have correct landmark roles (`role="navigation"`, `role="banner"`, `role="contentinfo"`)
- [ ] Mobile menu toggle has `aria-expanded` and `aria-controls` attributes
- [ ] Form controls with no visible label have `aria-label` or `aria-labelledby`
- [ ] Screen reader audit confirms all interactive elements are meaningfully announced

## Definition of Done
- [ ] Code implemented
- [ ] Tests passing
- [ ] Validation passing

## Implementation Notes
Audit interactive elements using axe-core or Lighthouse accessibility report. Prioritize icon-only controls and mobile menu toggle. Add ARIA attributes directly in HTML partials and page templates. Update `core.js` to dynamically manage `aria-expanded` state on the mobile menu.

## Rollback Notes
Remove added ARIA attributes from HTML files and revert `core.js` changes to the mobile menu toggle logic.
