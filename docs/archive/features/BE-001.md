# BE-001: Calendar Integration (Calendly)
- **Feature ID:** BE-001 | **Category:** Backend | **Priority:** HIGH | **Phase:** 08 | **Status:** Pending | **Est. Hours:** 2

## Business Goal
Enable visitors to schedule demos directly on the website, converting interest into qualified meetings without requiring email back-and-forth or manual coordination.

## Technical Goal
Replace the static HTML/CSS calendar mockup on the demo booking page with an embedded scheduling widget from Calendly or Cal.com.

## Problem Statement
The demo booking page displays a calendar UI built entirely with HTML and CSS. It has no functionality — users cannot select a date, pick a time slot, or complete a booking. The page creates an expectation of self-service scheduling that it fails to fulfill, likely causing drop-off.

## Current Behavior
`demo-booking.html` contains a purely decorative calendar built with styled `<div>` elements. No scheduling logic exists.

## Expected Behavior
The page embeds a functional scheduling widget (Calendly inline embed or Cal.com) that allows visitors to view available time slots, select one, provide their details, and receive a confirmation — all without leaving the site.

## Affected Pages / Components / Files
- `src/pages/demo-booking.html` — replace static calendar with widget embed
- `site.json` — add calendar provider configuration (e.g., Calendly username or Cal.com event URL)

## Dependencies
None (external: Calendly or Cal.com account required)

## Acceptance Criteria
- [ ] Static calendar HTML/CSS is replaced with an embedded scheduling widget
- [ ] Widget loads asynchronously and does not block page render
- [ ] Booking flow is functional end-to-end (select date → choose time → submit details → confirmation)
- [ ] Widget is responsive on mobile viewports
- [ ] Provider URL is configurable via `site.json` (not hardcoded)
- [ ] `npm run build && npm run validate` passes

## Definition of Done
- [ ] Code implemented
- [ ] Tests passing
- [ ] Validation passing

## Implementation Notes
Prefer Calendly's inline embed as it requires zero backend work — a `<script>` tag and a target `<div>`. Store the Calendly URL in `site.json` under a `calendar` key. Use Calendly's `data-url` attribute approach so the build script can inject the configured URL at assembly time. Add a `<noscript>` fallback linking to the external booking page.

## Rollback Notes
Restore the static calendar markup and remove the widget embed script.
