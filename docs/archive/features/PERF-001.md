# PERF-001: PWA Service Worker
- **Feature ID:** PERF-001 | **Category:** Performance | **Priority:** MEDIUM | **Phase:** 10 | **Status:** Pending | **Est. Hours:** 1.5

## Business Goal
Enable offline access and faster repeat visits, improving the experience for users on unreliable connections and reducing perceived load time.

## Technical Goal
Create a service worker that caches core assets (CSS, JS, images, key pages) for offline use, registered alongside the existing `manifest.json`.

## Problem Statement
The build pipeline generates a `manifest.json` for PWA installability, but no service worker file exists. Without a service worker, the site cannot cache assets for offline use, cannot provide a custom offline fallback page, and cannot take advantage of the PWA install prompt in supported browsers.

## Current Behavior
`manifest.json` is generated by the build script. No service worker is registered or served.

## Expected Behavior
A service worker (`sw.js`) is created, registered from `core.js`, and caches the application shell (CSS, JS, logo, key pages) on install. Subsequent navigations serve from cache, falling back to network with a basic offline page when unavailable.

## Affected Pages / Components / Files
- New file: `assets/js/sw.js` — service worker logic
- `assets/js/core.js` — service worker registration
- `scripts/assemble.mjs` — include `sw.js` in build output

## Dependencies
None

## Acceptance Criteria
- [ ] `sw.js` is served from the site root
- [ ] Service worker is registered on page load (no errors in console)
- [ ] Core assets (CSS, JS, logo, key fonts) are cached on first visit
- [ ] Returning visits load cached assets instantly
- [ ] Offline visits display a cached version of the page (or a custom offline fallback)
- [ ] Service worker has an update mechanism (cache versioning or stale-while-revalidate)
- [ ] `npm run build && npm run validate` passes

## Definition of Done
- [ ] Code implemented
- [ ] Tests passing
- [ ] Validation passing

## Implementation Notes
Use a cache-first strategy for static assets (CSS, JS, images) with versioned cache names to enable clean updates. Use network-first for HTML pages with an offline fallback. Keep the service worker simple — avoid complex routing unless proven necessary. Add the service worker file to the build output copy step in `assemble.mjs`.

## Rollback Notes
Remove `sw.js`, the registration call in `core.js`, and the build step that copies it. Unregister the service worker from any active clients if needed.
