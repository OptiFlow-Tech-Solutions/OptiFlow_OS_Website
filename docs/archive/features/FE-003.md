# FE-003: Cookie Consent Banner (GDPR/DPDP Act 2023)
- **Feature ID:** FE-003
- **Category:** Frontend
- **Parent Module:** Frontend
- **Priority:** HIGH
- **Phase:** 10
- **Status:** Pending
- **Est. Hours:** 1.5

## Business Goal
Achieve compliance with GDPR and the Indian Digital Personal Data Protection (DPDP) Act 2023 by providing users with transparent cookie consent options, reducing legal risk and building user trust.

## Technical Goal
Implement a cookie consent banner component that appears on first visit, captures user consent preferences, and conditionally loads tracking scripts based on user choice.

## Problem Statement
The site currently has no cookie consent mechanism. This violates GDPR requirements for EU visitors and the DPDP Act 2023 requirements for Indian users, exposing the business to regulatory penalties.

## Current Behavior
No cookie consent banner is displayed. All scripts, including any analytics or tracking, load unconditionally on every page visit.

## Expected Behavior
A cookie consent banner appears on the first visit (or until consent is given). It offers Accept All, Reject All, and Customize options. User preference is stored in `localStorage`. Accepted cookie categories trigger corresponding scripts; rejected categories block them.

## Affected Pages
- All 14 site pages (index, about, services, contact, careers, blog, pricing, faq, terms, privacy, portfolio, testimonials, 404, sitemap)

## Affected Components
- Cookie consent banner UI
- Consent preference manager
- Script loader gate

## Affected Files
- `src/partials/cookie-banner.html` (new)
- `assets/css/core.css`
- `assets/js/core.js`

## Dependencies
- none

## Acceptance Criteria
- [ ] Cookie consent banner displays on first visit across all pages
- [ ] Banner offers Accept All, Reject All, and Customize options
- [ ] User preference is persisted in `localStorage`
- [ ] Tracking/analytics scripts only load when consented
- [ ] Banner is dismissible and does not reappear after consent is given
- [ ] Banner is keyboard accessible and screen reader friendly
- [ ] Banner styles match the existing design system using CSS variables

## Definition of Done
- [ ] Code implemented
- [ ] Tests passing

## Implementation Notes
Create `src/partials/cookie-banner.html` with the banner HTML. Add banner styles to `core.css` using existing CSS variables for consistency. Add consent logic to `core.js`: check `localStorage` for existing consent, show/hide banner, gate script loading. Include the partial via `<!-- INCLUDE: cookie-banner -->` in the build pipeline.

## Rollback Notes
Remove `cookie-banner.html` partial, delete cookie consent styles from `core.css`, remove consent logic from `core.js`, and remove the include directive from the build pipeline.
