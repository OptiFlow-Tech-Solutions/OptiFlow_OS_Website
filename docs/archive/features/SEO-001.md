# SEO-001: Product Structured Data
- **Feature ID:** SEO-001 | **Category:** SEO | **Priority:** MEDIUM | **Phase:** 09 | **Status:** Pending | **Est. Hours:** 0.5

## Business Goal
Enhance search engine visibility by providing structured data that enables rich results (product carousels, software app cards), increasing click-through rates from SERPs.

## Technical Goal
Add `Product` and `SoftwareApplication` JSON-LD schemas to the build script's `injectJSONLD` function, populating them from `site.json` data.

## Problem Statement
The site currently generates only `Organization`, `WebSite`, and page-specific schemas. No `Product` or `SoftwareApplication` structured data exists, which means search engines cannot surface rich product information for queries like "logistics optimization software."

## Current Behavior
`assemble.mjs` `injectJSONLD` function generates Organization, WebSite, WebPage, and FAQ schemas but not Product or SoftwareApplication.

## Expected Behavior
`injectJSONLD` also outputs `Product` and `SoftwareApplication` JSON-LD blocks on the homepage (or relevant product pages), populated with product name, description, category, operating system, and application category from `site.json`.

## Affected Pages / Components / Files
- `scripts/assemble.mjs` — extend `injectJSONLD` function
- `site.json` — verify/ensure product metadata fields exist

## Dependencies
None

## Acceptance Criteria
- [ ] Homepage (or product page) includes valid `Product` JSON-LD schema
- [ ] Homepage (or product page) includes valid `SoftwareApplication` JSON-LD schema
- [ ] Both schemas pass Google Rich Results Test validation
- [ ] Schema values are sourced from `site.json`, not hardcoded
- [ ] `npm run build && npm run validate` passes

## Definition of Done
- [ ] Code implemented
- [ ] Tests passing
- [ ] Validation passing

## Implementation Notes
Add `product` and `softwareApplication` objects to `site.json` (or derive from existing fields). The `SoftwareApplication` schema should include `operatingSystem`, `applicationCategory`, and `offers`. The `Product` schema should include `name`, `description`, `category`, and `brand`. Both can share the same `aggregateRating` if reviews exist; otherwise, omit the rating to avoid invalid structured data.

## Rollback Notes
Remove the Product and SoftwareApplication schema blocks from `injectJSONLD` and any added fields from `site.json`.
