# SEO-003: Social Media OG Image
- **Feature ID:** SEO-003 | **Category:** SEO | **Priority:** LOW | **Phase:** 09 | **Status:** Pending | **Est. Hours:** 0.5

## Business Goal
Improve social media link previews with a dedicated share card image, increasing engagement when the site URL is shared on platforms like Twitter/X, LinkedIn, and Facebook.

## Technical Goal
Create a 1200x630px social share image and update the build pipeline to reference it in `og:image` meta tags.

## Problem Statement
The `og:image` meta tag currently points to the site logo, which is typically a small, square or horizontal mark unsuitable for the 1.91:1 aspect ratio expected by social platforms. This results in cropped, empty, or missing preview images when the site is shared.

## Current Behavior
`og:image` references the logo file. No dedicated social share card image (1200x630px) exists.

## Expected Behavior
A purpose-built 1200x630px social share image exists as an asset, and `og:image` points to it. Additional tags (`og:image:width`, `og:image:height`, `twitter:card`) are set appropriately.

## Affected Pages / Components / Files
- `scripts/assemble.mjs` — update `og:image` injection
- New asset: `assets/images/og-image.png` (or similar)

## Dependencies
None (requires creation of the OG image asset)

## Acceptance Criteria
- [ ] A 1200x630px social share image exists in the assets directory
- [ ] `og:image` meta tag references the new image, not the logo
- [ ] `og:image:width` and `og:image:height` meta tags are present with correct dimensions
- [ ] `twitter:card` is set to `summary_large_image`
- [ ] Image is included in the build output
- [ ] `npm run build && npm run validate` passes

## Definition of Done
- [ ] Code implemented
- [ ] Tests passing
- [ ] Validation passing

## Implementation Notes
The image should include the OptiFlow OS brand mark, a concise value proposition, and the site URL — all legible at thumbnail size. If a designer is unavailable, a simple HTML/CSS-to-image approach using a headless browser or a Node canvas library during build can auto-generate the card from data in `site.json`.

## Rollback Notes
Restore `og:image` to point to the logo and remove the social share image asset.
