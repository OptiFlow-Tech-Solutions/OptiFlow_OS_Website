# ACC-002: Skip-to-Content Link Consistency
- **Feature ID:** ACC-002
- **Category:** Accessibility
- **Parent Module:** Accessibility
- **Phase:** 05
- **Status:** Pending
- **Est. Hours:** 0.5

## Business Goal
Ensure keyboard and screen reader users can bypass repetitive navigation across every page, providing a consistent and efficient browsing experience.

## Technical Goal
Verify the skip-to-content link exists and functions correctly on all 14 pages, standardizing its implementation where missing or inconsistent.

## Problem Statement
A skip-to-content link is present on some pages but has not been verified across the full site. Inconsistent availability means keyboard users on some pages must tab through the entire navigation to reach main content.

## Current Behavior
Some pages include a skip-to-content link; others may not. The implementation may vary across pages that do have it.

## Expected Behavior
Every page includes an identical, functional skip-to-content link as the first focusable element. On activation (Enter key), focus moves to the `<main>` element, bypassing the navigation.

## Affected Pages
- All 14 site pages (index, about, services, contact, careers, blog, pricing, faq, terms, privacy, portfolio, testimonials, 404, sitemap)

## Affected Components
- Skip-to-content link component
- `<main>` element with `id="main-content"` and `tabindex="-1"`

## Affected Files
- `src/pages/*.html` (all 14 pages)
- `src/partials/nav.html`
- `assets/css/core.css`

## Dependencies
- none

## Acceptance Criteria
- [ ] All 14 pages include a skip-to-content link as the first focusable element
- [ ] Skip link targets `#main-content` with correct `tabindex="-1"` on the main element
- [ ] Visual focus ring is visible when the link receives keyboard focus
- [ ] Skip link is styled consistently across all pages

## Definition of Done
- [ ] Code implemented
- [ ] Tests passing
- [ ] Validation passing
- [ ] Documentation updated

## Implementation Notes
Add the skip-to-content link to `src/partials/nav.html` so it renders on every page via `<!-- INCLUDE: nav -->`. Ensure `core.css` has the skip-link styles (offscreen by default, visible on focus). Verify each page's `<main>` element has `id="main-content"` and `tabindex="-1"`.

## Rollback Notes
Remove the skip link from `nav.html` and revert any `id` or `tabindex` changes on `<main>` elements.
