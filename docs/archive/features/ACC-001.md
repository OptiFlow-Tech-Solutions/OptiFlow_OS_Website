# ACC-001: Add aria-hidden to Decorative SVGs
- **Feature ID:** ACC-001
- **Category:** Accessibility
- **Parent Module:** Accessibility
- **Priority:** MEDIUM
- **Phase:** 05
- **Status:** Pending
- **Est. Hours:** 1.5

## Business Goal
Eliminate screen reader noise from decorative SVG icons so that assistive technology users can navigate the site without being overwhelmed by redundant icon announcements.

## Technical Goal
Add `aria-hidden="true"` to all decorative/inline SVG icons across the codebase while ensuring meaningful SVGs retain proper accessible labels.

## Problem Statement
Hundreds of inline SVGs across all 14 pages, nav.html, and footer.html lack `aria-hidden="true"`, causing screen readers to announce each icon as a separate element and creating a confusing, noisy experience for users relying on assistive technology.

## Current Behavior
Decorative SVG icons (social media icons, chevrons, checkmarks, etc.) are exposed to the accessibility tree and announced by screen readers without conveying useful information.

## Expected Behavior
All decorative/inline SVG icons have `aria-hidden="true"` set, removing them from the accessibility tree. Meaningful SVGs (e.g., logos with text alternatives, informational icons) retain appropriate `aria-label` or `role="img"` attributes.

## Affected Pages
- All 14 site pages (index, about, services, contact, careers, blog, pricing, faq, terms, privacy, portfolio, testimonials, 404, sitemap)
- `src/partials/nav.html`
- `src/partials/footer.html`

## Affected Components
- Navigation icons
- Footer social media icons
- Section decorative icons
- Bullet/checkmark icons
- Arrow/chevron icons
- Card accent icons

## Affected Files

## Definition of Done

## Implementation Notes
Audit all SVG usage across the codebase. Batch-add `aria-hidden="true"` to decorative SVGs via the build pipeline (`assemble.mjs`) where possible. For SVGs that convey meaning, add `role="img"` and `aria-label` instead. Use a grep-based audit to identify SVGs missing aria attributes.

## Rollback Notes
Revert the grep/replace operation in `assemble.mjs`. If pages were individually edited, revert affected page files.
